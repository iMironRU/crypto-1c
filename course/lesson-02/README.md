# Урок 02. Хеширование: необратимый отпечаток

> ⏱ Время: ~45 минут  
> Что нужно до начала: Урок 01, базовое понимание что такое строки и файлы в 1С  
> Что будет после: умеете вычислять хеш строки и файла, понимаете зачем это нужно

## Зачем это знать

Вы скачали обновление конфигурации с сайта вендора. Рядом с файлом указан SHA256:
`3b4c5d...`. Как убедиться что файл не подменили и не повредили при передаче?
Вычислить хеш скачанного файла и сравнить с опубликованным.

Или другой сценарий: система обмена данными между двумя базами 1С.
Как убедиться что XML-пакет дошёл без изменений? Отправляете хеш вместе с пакетом.
Получатель считает хеш заново и сравнивает.

Хеш — это отпечаток данных. Как отпечаток пальца: практически уникален для каждого
человека (теоретически двое могут дать одинаковый, но это крайне редко),
всегда одинаковый для одного и того же человека, и по отпечатку нельзя восстановить
самого человека. Именно эти три свойства делают хеш полезным инструментом.

## Как это работает

Хеш-функция принимает данные любого размера — строку в один символ или файл в несколько
гигабайт — и возвращает строку фиксированной длины. Для SHA256 это всегда 256 бит,
или 32 байта, или 64 HEX-символа, или 44 символа в Base64.

Три ключевых свойства:

**Детерминированность.** Одни и те же данные → всегда один и тот же хеш.
Хеш строки «Привет» будет одинаковым хоть сегодня, хоть через год.

**Необратимость.** По хешу нельзя восстановить исходные данные.
Хеш-функция — математическая операция «в одну сторону».
Именно поэтому хеши используют для хранения паролей: даже если база данных утечёт,
пароли по хешам не восстановить.

**Лавинный эффект.** Изменение одного байта в исходных данных полностью меняет хеш.
«Привет» и «привет» (строчная «п») дадут совершенно разные хеши — ни одного
совпадающего символа.

### Про MD5 — не используйте

MD5 — устаревший алгоритм. Для него известны способы создать два разных файла
с одинаковым хешем (коллизии). В новых проектах используйте SHA256 или SHA512.
MD5 допустим только там где уже есть унаследованная система, которую нельзя изменить.

## Пример 1. Хеш строки

Самый простой случай. Важно: сначала строка превращается в байты,
и только потом байты уходят в хеш-функцию.

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows

Попытка

    ТекстДляХеша = "Привет, мир!";

    // Шаг 1. Строка → байты
    // Кодировка UTF-8 — стандарт. Одна и та же строка в разных
    // кодировках даст разные байты и разные хеши. Фиксируйте кодировку.
    ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТекстДляХеша, "UTF-8");

    // Шаг 2. Считаем хеш
    Хеширование = Новый ХешированиеДанных(ХешФункция.SHA256);
    Хеширование.Добавить(ДвоичныеДанные);
    ХешДвоичные = Хеширование.ХешСумма; // результат — ДвоичныеДанные, 32 байта

    // Шаг 3. Байты → строка для хранения и вывода
    // Base64: 44 символа, компактно, удобно хранить в строковом реквизите
    ХешBase64 = Base64Строка(ХешДвоичные);
    Сообщить("SHA256 (Base64): " + ХешBase64);
    // SHA256 (Base64): fnCFinMkqFpOWTnQDEHZW6kEMEsq6tCyEuM2s8n0Lmc=

Исключение
    Сообщить("Ошибка хеширования: " + ОписаниеОшибки());
КонецПопытки;
```

Обратите внимание: если убрать `"UTF-8"` или заменить на `"Windows-1251"` —
хеш будет другим. Всегда явно указывайте кодировку.

### Два формата результата: Base64 и HEX

Хеш — это байты. Их нужно как-то представить в виде текста. Два распространённых способа:

| Формат | SHA256 длина | Когда использовать |
|--------|-------------|-------------------|
| Base64 | 44 символа  | Хранение в базе, передача в JSON/HTTP |
| HEX    | 64 символа  | Сравнение с утилитами (certmgr, openssl), логи |

Перевод хеша в HEX — чуть сложнее, потому что в 1С нет встроенной функции.
Вот готовый вспомогательный фрагмент:

```bsl
// Вспомогательная функция: ДвоичныеДанные → HEX-строка
Функция ДвоичныеВHex(ДвоичныеДанные)

    // Копируем байты в буфер для поштучного чтения
    Буфер = Новый БуферДвоичныхДанных(ДвоичныеДанные.Размер());
    ДвоичныеДанные.КопироватьВБуфер(Буфер, 0);

    Результат = "";
    Для й = 0 По Буфер.Размер() - 1 Цикл
        // Каждый байт → два hex-символа. Прав(..., 2) обеспечивает ведущий ноль
        Результат = Результат + Прав("0" + НРег(Формат(Буфер.ПолучитьБайт(й), "ЧС=16")), 2);
    КонецЦикла;

    Возврат Результат;

КонецФункции;

// Использование:
ХешHex = ДвоичныеВHex(ХешДвоичные);
Сообщить("SHA256 (HEX): " + ХешHex);
// SHA256 (HEX): 7ec4098a73249a85a693959d40441d9ba904304b2aeac4b21ae336b3c9f42d97
```

## Пример 2. Хеш файла

Хеш файла — та же схема. Разница только в первом шаге:
файл читается как `ДвоичныеДанные` напрямую, без промежуточной конвертации строк.

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows
// - Файл должен существовать по указанному пути

Попытка

    ПутьКФайлу = "C:\temp\document.pdf"; // замените на реальный путь

    // Шаг 1. Читаем файл как байты
    ФайлДвоичные = Новый ДвоичныеДанные(ПутьКФайлу);

    // Шаг 2. Считаем хеш
    Хеширование = Новый ХешированиеДанных(ХешФункция.SHA256);
    Хеширование.Добавить(ФайлДвоичные);
    ХешДвоичные = Хеширование.ХешСумма;

    // Шаг 3. Байты → HEX (файловые контрольные суммы принято показывать в HEX)
    ХешHex = ДвоичныеВHex(ХешДвоичные); // функция из примера выше
    Сообщить("SHA256: " + ХешHex);

Исключение
    Сообщить("Ошибка: " + ОписаниеОшибки());
КонецПопытки;
```

> ℹ️ `ХешированиеДанных` умеет принимать данные по частям через несколько
> вызовов `Добавить()`. Это удобно для больших файлов: читайте файл кусками
> и добавляйте каждый кусок — хеш получится тот же что от всего файла целиком.

## Пример 3. Проверка целостности файла

Применим на практике: сравниваем хеш скачанного файла с опубликованной контрольной суммой.

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows

Попытка

    ПутьКФайлу   = "C:\temp\update.cf";
    ОжидаемыйХеш = "3b4c5d6e7f..."; // хеш с сайта вендора, в HEX

    // Считаем хеш скачанного файла
    ФайлДвоичные = Новый ДвоичныеДанные(ПутьКФайлу);
    Хеширование  = Новый ХешированиеДанных(ХешФункция.SHA256);
    Хеширование.Добавить(ФайлДвоичные);
    ФактическийХеш = ДвоичныеВHex(Хеширование.ХешСумма); // функция из примера выше

    // Сравниваем без учёта регистра — хеши могут быть опубликованы в разном регистре
    Если НРег(СокрЛП(ФактическийХеш)) = НРег(СокрЛП(ОжидаемыйХеш)) Тогда
        Сообщить("Файл целостен. Можно устанавливать.");
    Иначе
        Сообщить("Ошибка! Хеши не совпадают. Файл повреждён или подменён.");
    КонецЕсли;

Исключение
    Сообщить("Ошибка: " + ОписаниеОшибки());
КонецПопытки;
```

## Где применяется в реальной работе

**Проверка целостности файлов.** Вендор публикует SHA256 дистрибутива.
Скачали — посчитали хеш — сравнили. Совпало — файл не тронут.

**Обмен данными между базами.** В пакет добавляете хеш его содержимого.
Принимающая сторона считает хеш заново и сравнивает.
Любое искажение при передаче сразу видно.

**Хранение паролей.** Вместо пароля в базе хранится его хеш.
При входе пользователь вводит пароль, вы хешируете его и сравниваете с хранимым.
Пароль в базе никогда не оказывается в открытом виде. Подробно — в Уроке 03.

**Дедупликация.** Хеш файла — его уникальный идентификатор.
Если хеши двух файлов совпадают — содержимое одинаковое, можно хранить один экземпляр.

## Что запомнить

- Хеш-функция — три свойства: **детерминированная** (одинаковый вход → одинаковый выход),
  **необратимая** (по хешу нельзя восстановить данные), **лавинный эффект** (1 байт → другой хеш).
- SHA256 — современный стандарт. MD5 взломан, в новых проектах не использовать.
- Полный путь: `Строка → ДвоичныеДанные → ХешированиеДанных → ДвоичныеДанные → Base64 или HEX`.
- Кодировка важна: UTF-8 и Windows-1251 дают разные байты и разные хеши. Всегда фиксируйте явно.
- `ХешированиеДанных` — встроенный объект платформы, не требует установки дополнительного ПО.
