# Урок 06. Электронная подпись: цифровая печать

> ⏱ Время: ~45 минут  
> Что нужно до начала: Уроки 01–05 (особенно урок 04 — ключи и сертификаты)  
> Что будет после: умеете подписывать документы и проверять подпись в 1С

## Зачем это знать

Задача: отправить акт сверки контрагенту в электронном виде.
Контрагент должен быть уверен в двух вещах: что документ пришёл именно от вас,
и что по дороге никто его не изменил.

Печать на бумажном договоре решает обе задачи: она уникальна (подделать трудно)
и нанесена на конкретный лист бумаги (если лист подменить — печать не совпадёт).

Электронная подпись — цифровой аналог печати. Она привязывает документ
к конкретному человеку (через его закрытый ключ) и фиксирует содержимое
(изменится хоть один байт — проверка подписи с высокой вероятностью не пройдёт).

В отличие от шифрования (урок 05), ЭП не скрывает содержимое — документ
остаётся читаемым. ЭП отвечает на другой вопрос: «кто и что именно подписал?»

## Как это работает

### Шаг 1: хешируем, не подписываем весь документ

Подписывать большой файл целиком асимметричным алгоритмом — долго.
Поэтому подписывают не сам документ, а его **хеш**.

На бумаге вы не переносите весь договор в журнал регистрации — вы записываете
номер, дату и краткое описание. В ЭП роль такого отпечатка выполняет хеш документа.

Хеш имеет фиксированную длину, зависящую от алгоритма (SHA-256 — 32 байта,
SHA-1 — 20 байт), вычисляется быстро и практически однозначно идентифицирует
содержимое (урок 02). Изменится хоть один байт документа — хеш станет другим.

### Шаг 2: хеш + закрытый ключ = подпись

Подписант вычисляет значение подписи для хеша документа с помощью **закрытого ключа**.
Это и есть подпись. Математически это не «шифрование хеша» — для ГОСТ Р 34.10
операция принципиально другая, хотя для RSA аналогия похожа.

```
Документ → хеш(алгоритм провайдера) → формирование подписи закрытым ключом → Подпись
```

Без доступа к закрытому ключу создать корректную подпись практически невозможно.

### Шаг 3: проверка подписи получателем

Получатель делает два действия:

1. Вычисляет хеш полученного документа самостоятельно.
2. Проверяет подпись открытым ключом подписанта (он в сертификате).
   Платформа убеждается, что подпись соответствует этому хешу и этому ключу.

```
Документ → хеш(алгоритм) → Хеш документа
Подпись + Хеш → проверка открытым ключом → ✓ верна / ✗ нет
```

Если проверка прошла — данные соответствуют подписи, а подпись создана закрытым ключом,
связанным с сертификатом. Если не прошла — документ изменился или подпись поддельная.

> Доверие к личности подписанта требует отдельной проверки сертификата:
> цепочки доверия, срока действия и назначения ключа.

### Три способа хранить подпись

| Тип | Как хранится | Читаем ли оригинал без крипто-ПО |
|-----|-------------|----------------------------------|
| **Отсоединённая** | Подпись — отдельный файл `.sig`/`.p7s` | Да |
| **Присоединённая (CMS)** | Документ упакован в CMS-конверт | Нет — нужен CMS-инструмент |
| **Встроенная в формат** | Подпись внутри структуры PDF/DOCX/XML | Да |

**Отсоединённая (detached)** — подпись хранится отдельно, документ не меняется.
Для проверки нужны оба файла. Распространённый формат при обмене с госсистемами.
В 1С `МенеджерКрипто.Подписать()` создаёт именно этот тип.

**Присоединённая CMS (attached)** — документ целиком упаковывается в CMS-контейнер
(`.p7m`). Исходный файл больше не является самостоятельным PDF или DOCX —
его нельзя открыть в Adobe Reader или Word. Для чтения нужен инструмент,
понимающий CMS (например, КриптоАРМ). Используется реже.

**Встроенная в формат** — подпись внедряется в структуру самого файла.
PDF остаётся PDF: его открывает любой читалка, Adobe показывает значок подписи.
DOCX остаётся DOCX. В 1С это реализуется через `ЗаписьPDF.ЗаписатьПодпись()`
(см. Пример 5).

### Три вида ЭП в российском праве

| Вид | Что это | Юридическая сила |
|-----|---------|-----------------|
| ПЭП (простая) | Логин/пароль, SMS-код, галочка «согласен» | По соглашению сторон |
| НЭП (неквалифицированная) | Криптографическая, сертификат от любого УЦ | По соглашению сторон |
| КЭП (квалифицированная) | Криптографическая, сертификат аккредитованного УЦ, ГОСТ | Признаётся равнозначной собственноручной подписи в предусмотренных законом случаях |

**УЦ** — удостоверяющий центр, организация, которая проверяет личность и выпускает сертификаты.

В учебных примерах используем Microsoft Strong CSP и RSA.
Для ФНС, ФСС, Госуслуг нужна **КЭП**: КриптоПро + сертификат аккредитованного УЦ.

## Пример 1. Подписать данные

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows
// - Сертификат с закрытым ключом в личном хранилище (см. docs/certificates.md)
// - Замените Отпечаток на ваше значение из урока 04

Попытка

    Отпечаток = "A1B2C3D4E5F6..."; // вставьте ваш отпечаток

    МенеджерКрипто = Новый МенеджерКриптографии(
        "Microsoft Strong Cryptographic Provider", "", 1
    );

    // Получаем сертификат с закрытым ключом
    Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);

    // Шаг 1. Строка → ДвоичныеДанные (UTF-8)
    ТекстДокумента = "Акт сверки №123 от 01.01.2025. Итого: 150 000 руб.";
    ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТекстДокумента, "UTF-8");

    // Шаг 2. Подписать: платформа вычисляет хеш (алгоритм зависит от провайдера:
    // Microsoft Strong CSP — SHA-1; тип 24 (Enhanced RSA and AES CSP) — SHA-256;
    // КриптоПро — ГОСТ Р 34.11-2012).
    // Возвращает байты подписи в формате CMS (отсоединённая — исходные данные не включены).
    Подпись = МенеджерКрипто.Подписать(ДвоичныеДанные, Сертификат);

    // Шаг 3. Для хранения или передачи → переводим в Base64
    ПодписьBase64 = Base64Строка(Подпись);

    // Для хранения документа в базе тоже нужен Base64 от тех же байтов UTF-8
    ДанныеДокументаBase64 = Base64Строка(ДвоичныеДанные);

    Сообщить("Подписано успешно.");
    Сообщить("Размер подписи (байт): " + Подпись.Размер());
    Сообщить("Подпись Base64 (первые 80 символов): " + Лев(ПодписьBase64, 80) + "...");
    // В базе хранят пару: ДанныеДокументаBase64 + ПодписьBase64

Исключение
    Сообщить("Ошибка при подписании: " + ОписаниеОшибки());
    // Частые причины:
    // - сертификат найден, но закрытого ключа нет (ключ на другом компьютере)
    // - нет прав доступа к закрытому ключу
    // - сертификат не предназначен для подписи (нет признака DigitalSignature)
КонецПопытки;
```

> **В боевой системе (КЭП / ГОСТ):** замените инициализацию провайдера:
> ```bsl
> МенеджерКрипто = Новый МенеджерКриптографии(
>     "Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider", "", 80
> );
> ```
> Точный тип провайдера зависит от версии КриптоПро — проверьте через Пример 1 урока 04.
> Методы `Подписать`, `ПроверитьПодпись` и `НачатьПроверкуПодписи` работают без изменений.

## Пример 2. Проверить подпись

```bsl
// Сценарий: есть текст документа и подпись к нему (в Base64).
// Проверяем: не изменён ли документ и верна ли подпись.

// Требования:
// - Платформа 1С 8.3.6+
// - Windows
// - Сертификат подписанта не нужно устанавливать отдельно:
//   он встроен в CMS-пакет подписи

Попытка

    МенеджерКрипто = Новый МенеджерКриптографии(
        "Microsoft Strong Cryptographic Provider", "", 1
    );
    // Для ГОСТ: замените инициализацию провайдера как в Примере 1

    // Восстанавливаем данные и подпись из Base64
    ТекстДокумента = "Акт сверки №123 от 01.01.2025. Итого: 150 000 руб.";
    ПодписьBase64  = "..."; // вставьте Base64 из Примера 1

    // Шаг 1. Строка → ДвоичныеДанные (та же кодировка, что при подписании)
    ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТекстДокумента, "UTF-8");
    // Шаг 2. Base64 → ДвоичныеДанные подписи
    Подпись = Base64Значение(ПодписьBase64);

    // Шаг 3. Проверить подпись: платформа вычисляет хеш документа,
    // проверяет его соответствие подписи через открытый ключ из CMS-пакета.
    // Сертификат подписанта встроен в подпись — указывать его отдельно не нужно.
    // Бросает исключение если проверка не прошла, ничего не возвращает если прошла.
    МенеджерКрипто.ПроверитьПодпись(ДвоичныеДанные, Подпись);

    Сообщить("Подпись криптографически корректна. Документ не изменён с момента подписания.");
    // Это подтверждает соответствие подписи данным, но не доверие к личности подписанта.
    // Для полного доверия проверьте цепочку сертификатов и его срок действия.

Исключение
    Сообщить("Подпись не прошла проверку: " + ОписаниеОшибки());
    // Частые причины:
    // - документ изменился после подписания
    // - подпись сформирована другим ключом
    // - несоответствие провайдера, алгоритма или формата CMS
КонецПопытки;
```

Попробуйте изменить хоть один символ в `ТекстДокумента` и запустить проверку —
получите ошибку. Это ключевое свойство ЭП: она криптографически привязана
к конкретным байтам документа.

> **В боевой системе (КЭП / ГОСТ):** замените инициализацию провайдера как в Примере 1.

## Пример 3. Подписать файл и сохранить подпись отдельно

При обмене с госсистемами подпись обычно хранят в отдельном файле `.sig` или `.p7s`.

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows
// - Сертификат с закрытым ключом в личном хранилище
// - Файл C:\Temp\Договор.pdf должен существовать
// - Замените Отпечаток на ваш

Попытка

    Отпечаток = "A1B2C3D4E5F6..."; // вставьте ваш отпечаток

    МенеджерКрипто = Новый МенеджерКриптографии(
        "Microsoft Strong Cryptographic Provider", "", 1
    );
    // Для ГОСТ: замените инициализацию провайдера как в Примере 1

    Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);

    // Шаг 1. Читаем файл → ДвоичныеДанные
    ПутьКФайлу = "C:\Temp\Договор.pdf";
    ДвоичныеДанные = Новый ДвоичныеДанные(ПутьКФайлу);

    // Шаг 2. Подписать (отсоединённая подпись — файл не изменяется)
    Подпись = МенеджерКрипто.Подписать(ДвоичныеДанные, Сертификат);

    // Шаг 3. Сохранить подпись рядом с файлом
    ПутьПодписи = ПутьКФайлу + ".sig";
    Подпись.Записать(ПутьПодписи);

    Сообщить("Файл подписан.");
    Сообщить("Документ: " + ПутьКФайлу);
    Сообщить("Подпись:  " + ПутьПодписи);
    Сообщить("Размер подписи (байт): " + Подпись.Размер());

Исключение
    Сообщить("Ошибка: " + ОписаниеОшибки());
КонецПопытки;
```

Для проверки такой подписи нужны оба файла: `Договор.pdf` и `Договор.pdf.sig`.
Код проверки — как в Примере 2, только `ДвоичныеДанные` читаются из файла:
`ДвоичныеДанные = Новый ДвоичныеДанные(ПутьКФайлу);`

> **В боевой системе (КЭП / ГОСТ):** замените инициализацию провайдера как в Примере 1.

## Пример 4. Получить информацию о подписанте из подписи

CMS-пакет подписи содержит сертификат подписанта — его можно извлечь и показать пользователю.

```bsl
// Требования:
// - Платформа 1С 8.3.6+
// - Windows
// - ПодписьBase64 из Примера 1 (или 3)

Попытка

    МенеджерКрипто = Новый МенеджерКриптографии(
        "Microsoft Strong Cryptographic Provider", "", 1
    );
    // Для ГОСТ: замените инициализацию провайдера как в Примере 1

    ПодписьBase64 = "..."; // вставьте Base64 из Примера 1
    Подпись = Base64Значение(ПодписьBase64);

    // НачатьПроверкуПодписи — разобрать CMS-пакет без проверки соответствия документу.
    // Извлекает список подписантов и их сертификаты из пакета.
    // Для проверки корректности подписи используйте ПроверитьПодпись (Пример 2).
    ДанныеПодписи = МенеджерКрипто.НачатьПроверкуПодписи(Подпись);

    Для Каждого Подписант Из ДанныеПодписи.Подписанты Цикл
        Сертификат = Подписант.Сертификат;
        Сообщить("---");
        Сообщить("Подписал:  " + Сертификат.Субъект);
        Сообщить("Издатель:  " + Сертификат.Издатель);
        Сообщить("Сертификат действует до: "
            + Формат(Сертификат.ДатаОкончания, "ДЛФ=Д"));
        // Время подписания берётся из атрибута CMS (signingTime).
        // Это не доверенное время — оно указано самим подписантом.
        // Для юридически значимого времени нужен штамп времени от службы TSA.
        Если ЗначениеЗаполнено(Подписант.ВремяПодписания) Тогда
            Сообщить("Время подписания (из CMS): "
                + Формат(Подписант.ВремяПодписания, "ДЛФ=ДВ"));
        КонецЕсли;
    КонецЦикла;

Исключение
    Сообщить("Ошибка: " + ОписаниеОшибки());
КонецПопытки;
```

> **В боевой системе (КЭП / ГОСТ):** замените инициализацию провайдера как в Примере 1.

## Пример 5. Встроить подпись прямо в PDF

Самый удобный вариант для пользователя: PDF остаётся PDF, подпись видна
в Adobe Reader как штамп, документ открывается без дополнительного ПО.

Платформа реализует это через объект `ЗаписьPDF` и `ОписаниеПодписиPDF`.

```bsl
// Требования:
// - Платформа 1С 8.3.19+
// - Windows, установлен КриптоПро CSP
// - Файл PDF должен существовать по ПутьКФайлуPDF
// - Сертификат с закрытым ключом в личном хранилище (замените Отпечаток)
// - В макете отчёта должен быть макет "ПодписьЭЦП" — визуальный штамп подписи

&НаКлиенте
Процедура ДобавитьПодписьPDF(Команда)

    Попытка

        // Тип 80 — КриптоПро для ГОСТ Р 34.10-2012.
        // Криптографическую операцию (хеш + подпись байтов PDF) выполняет провайдер;
        // встраивание результата в структуру файла берёт на себя платформа.
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider", "", 80
        );

        // Получаем сертификат с закрытым ключом по отпечатку
        Отпечаток = "A1B2C3D4E5F6..."; // вставьте ваш отпечаток
        Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);

        // Если ключ защищён паролем (не на токене с автоматическим PIN)
        МенеджерКрипто.ПарольДоступаКЗакрытомуКлючу = ПарольЭЦП;

        // Описание подписи: имя поля, сертификат, тип и поведение
        ПодписьPDF = Новый ОписаниеПодписиPDF;
        ПодписьPDF.Имя        = "ПодписьДоговора";
        ПодписьPDF.Сертификат = Сертификат; // явно указываем, чем подписываем

        // Утверждающая — обычная подпись содержимого документа.
        // Удостоверяющая — первая в цепочке, задаёт политику последующих.
        ПодписьPDF.ТипПодписи = ТипПодписиPDF.Утверждающая;

        // Заблокировать документ от изменений после подписания
        ПодписьPDF.БлокировкаДокумента = Истина;

        // ЗаписьPDF открывает существующий PDF-файл для встройки подписи.
        // PDF — это набор байтов; платформа вычисляет хеш этих байтов,
        // формирует подпись закрытым ключом и встраивает всё в структуру файла.
        ЗаписьPDF = Новый ЗаписьPDF(ПутьКФайлуPDF);

        // Штамп — видимый значок подписи на странице (аналог синей печати).
        // Макет "ПодписьЭЦП" — табличный документ с ФИО, датой, организацией.
        // Без штампа подпись криптографически верна, но визуально не заметна.
        ОтчетОбъект = РеквизитФормыВЗначение("Отчет");
        Штамп = ОтчетОбъект.ПолучитьМакет("ПодписьЭЦП");

        ЗаписьPDF.ЗаписатьПодпись(МенеджерКрипто, ПодписьPDF, Штамп);

        Сообщить("Подпись добавлена в файл: " + ПутьКФайлуPDF);
        // PDF открывается в любом просмотрщике;
        // Adobe Reader и Foxit показывают панель со статусом подписи.

    Исключение
        Сообщить("Ошибка при добавлении подписи в PDF: " + ОписаниеОшибки());
    КонецПопытки;

КонецПроцедуры
```

**Чем отличается от отсоединённой подписи (Примеры 1–3):**
- Исходный файл изменяется — подпись встраивается внутрь.
- PDF остаётся валидным PDF, открывается в любом просмотрщике.
- Adobe Reader показывает панель со статусом подписи и данными подписанта.
- Для проверки не нужен отдельный `.sig` файл.

**Визуальный штамп** — не декорация: это изображение, которое пользователь
видит на странице (аналог синей печати на бумаге). Макет готовят как
обычный табличный документ в конфигураторе.

## Что запомнить

- **ЭП не шифрует документ** — содержимое остаётся читаемым. ЭП отвечает
  на вопрос «кто подписал и не изменился ли документ?»
- **Подписывается хеш**, а не весь документ — быстро и однозначно привязывает
  подпись к содержимому. Операция создания подписи — не «шифрование хеша».
- **Создаётся закрытым ключом** (только у владельца), **проверяется открытым**
  (у любого, кто хочет проверить). Это противоположно шифрованию.
- **Три типа хранения подписи:** отсоединённая (`.sig` рядом с файлом),
  присоединённая CMS (документ упакован в контейнер, исходный просмотрщик его не откроет),
  встроенная в формат (PDF/DOCX остаётся читаемым — самый удобный для пользователя вариант).
- **Встроить подпись в PDF** в 1С — `ЗаписьPDF.ЗаписатьПодпись()`. Платформа 8.3.19+.
- **КЭП** — в типовой российской практике: КриптоПро, ГОСТ-алгоритмы и сертификат
  аккредитованного УЦ. В предусмотренных законом случаях признаётся равнозначной
  собственноручной подписи.
