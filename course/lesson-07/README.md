# Урок 07. Криптография в реальных задачах

> ⏱ Время: ~45 минут  
> Что нужно до начала: Уроки 01–06  
> Что будет после: умеете применять хеширование, шифрование и ЭП в типовых задачах 1С

## Зачем это знать

Представьте строителя, который умеет работать молотком, дрелью и пилой —
но не знает, что применить, когда нужно повесить полку. Знание инструментов
и умение их применять — разные вещи.

С криптографией то же самое. Уроки 02–06 объяснили инструменты по отдельности.
Теперь — реальные задачи, где нужно выбрать правильный инструмент
или скомбинировать несколько: хеш для контроля целостности,
шифрование для хранения секретов, подпись + шифрование для защищённой передачи.

Этот урок — справочник паттернов. Каждый раздел — отдельная задача с готовым кодом.

## Задача 1. Контроль целостности файлов

**Когда нужно:** файл передаётся между системами, и нужно убедиться
что он не повредился и не был подменён.

**Инструмент:** хеш SHA-256 (урок 02).

**Схема:** при сохранении вычислить хеш и сохранить рядом с файлом.
При получении — вычислить хеш снова и сравнить.

```bsl
// Шаг 1: вычислить хеш файла
Функция ХешФайла(ПутьКФайлу)

    Попытка
        ДвоичныеДанные = Новый ДвоичныеДанные(ПутьКФайлу);

        ХешированиеДанных = Новый ХешированиеДанных(ХешФункция.SHA256);
        ХешированиеДанных.Добавить(ДвоичныеДанные);

        Возврат НРег(ХешированиеДанных.ХешСумма);
    Исключение
        ВызватьИсключение "Ошибка при вычислении хеша файла: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции

// Шаг 2: при получении — проверить
Процедура ПроверитьЦелостностьФайла(ПутьКФайлу, ОжидаемыйХеш)

    ФактическийХеш = ХешФайла(ПутьКФайлу);

    Если ФактическийХеш <> НРег(ОжидаемыйХеш) Тогда
        ВызватьИсключение "Файл повреждён или подменён."
            + " Ожидался хеш: " + ОжидаемыйХеш
            + ", получен: "     + ФактическийХеш;
    КонецЕсли;

    Сообщить("Файл " + ПутьКФайлу + " прошёл проверку целостности.");

КонецПроцедуры

// Пример использования
// При сохранении:
ПутьКФайлу = "C:\Temp\Договор.pdf";
ХешПриСохранении = ХешФайла(ПутьКФайлу);
// Сохраняем ХешПриСохранении в базу вместе с реквизитами файла

// При получении или проверке:
ПроверитьЦелостностьФайла(ПутьКФайлу, ХешПриСохранении);
```

## Задача 2. Безопасное хранение токенов в базе

**Когда нужно:** в базе хранятся токены, пароли к внешним API, секретные ключи.
Если кто-то получит файл базы — он не должен их прочитать.

**Инструмент:** шифрование на сертификате (урок 05).

**Схема:** заводится технический сертификат (например, на сервере 1С).
При записи токен шифруется открытым ключом этого сертификата.
При чтении — расшифровывается закрытым ключом (ключ доступен только на сервере).

```bsl
// Требования:
// - Сертификат "TechCert" установлен в личном хранилище на сервере 1С
// - Замените Отпечаток на отпечаток технического сертификата

ОтпечатокТехСертификата = "A1B2C3D4E5F6..."; // Замените на реальный отпечаток

// Зашифровать токен для хранения в базе
Функция ЗашифроватьТокен(ТокенСтрока)

    Попытка
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Microsoft Strong Cryptographic Provider", "", 1
        );

        Сертификат = МенеджерКрипто.ПолучитьСертификат(ОтпечатокТехСертификата);

        ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТокенСтрока, "UTF-8");

        СписокСертификатов = Новый Массив;
        СписокСертификатов.Добавить(Сертификат);

        Зашифровано = МенеджерКрипто.Зашифровать(ДвоичныеДанные, СписокСертификатов);

        Возврат Base64Строка(Зашифровано); // сохраняем в строковый реквизит базы
    Исключение
        ВызватьИсключение "Ошибка при шифровании токена: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции

// Расшифровать токен при использовании
Функция РасшифроватьТокен(ЗашифрованноеBase64)

    Попытка
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Microsoft Strong Cryptographic Provider", "", 1
        );

        Зашифровано = Base64Значение(ЗашифрованноеBase64);
        ДвоичныеДанные = МенеджерКрипто.Расшифровать(Зашифровано);

        Возврат ПолучитьСтрокуИзДвоичныхДанных(ДвоичныеДанные, "UTF-8");
    Исключение
        ВызватьИсключение "Ошибка при расшифровании токена: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции

// Пример использования
ТокенAPI = "sk-live-abc123xyz789";
ЗашифрованныйТокен = ЗашифроватьТокен(ТокенAPI);
// Записываем ЗашифрованныйТокен в реквизит справочника "НастройкиAPI"

// При запросе к внешнему API:
ЗашифрованноеИзБазы = "..."; // читаем из реквизита
ЖивойТокен = РасшифроватьТокен(ЗашифрованноеИзБазы);
// Используем ЖивойТокен в HTTP-запросе, после использования он уходит из памяти
```

> **Важно:** не путайте с хешированием паролей (урок 03).
> Хранение паролей пользователей — хеш+соль. Хранение токенов и секретов,
> которые нужно обратно расшифровать — шифрование.

> **В боевой системе:** замените `Microsoft Strong Cryptographic Provider` (тип `1`)
> на `Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider` (тип `80`).

## Задача 3. Проверить сертификат перед использованием

**Когда нужно:** перед подписанием документа убедиться, что сертификат
ещё действителен. Истёкший сертификат выдаст ошибку при подписании
(или, хуже, молча создаст невалидную подпись в некоторых конфигурациях).

```bsl
// Проверить, что сертификат действителен по датам.
// Важно: функция проверяет только срок. Полная проверка (включая цепочку доверия
// и списки отозванных сертификатов) — через МенеджерКрипто.ПроверитьСертификат().
Функция СертификатДействителен(Сертификат)

    Сейчас = ТекущаяДата();

    Если Сейчас < Сертификат.ДатаНачала Тогда
        Возврат Ложь; // сертификат ещё не начал действовать
    КонецЕсли;

    Если Сейчас > Сертификат.ДатаОкончания Тогда
        Возврат Ложь; // сертификат истёк
    КонецЕсли;

    // Дополнительно: предупредить если осталось меньше 30 дней
    ДнейОсталось = (Сертификат.ДатаОкончания - Сейчас) / 86400;
    Если ДнейОсталось < 30 Тогда
        Сообщить("Предупреждение: сертификат истекает через "
            + Цел(ДнейОсталось) + " дн. (" 
            + Формат(Сертификат.ДатаОкончания, "ДЛФ=Д") + ")");
    КонецЕсли;

    Возврат Истина;

КонецФункции

// Пример использования перед подписанием
Функция ПодписатьДокументСПроверкой(ДвоичныеДанные, Отпечаток)

    Попытка
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Microsoft Strong Cryptographic Provider", "", 1
        );

        Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);

        Если НЕ СертификатДействителен(Сертификат) Тогда
            ВызватьИсключение "Сертификат недействителен: "
                + Формат(Сертификат.ДатаОкончания, "ДЛФ=Д");
        КонецЕсли;

        Подпись = МенеджерКрипто.Подписать(ДвоичныеДанные, Сертификат);
        Возврат Base64Строка(Подпись);
    Исключение
        ВызватьИсключение "Ошибка при подписании: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции
```

> **В боевой системе:** замените `Microsoft Strong Cryptographic Provider` (тип `1`)
> на `Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider` (тип `80`).

## Задача 4. Подписать и зашифровать документ для отправки

**Когда нужно:** отправить контрагенту документ, который:
- сможет прочитать только он (шифрование);
- докажет что отправили именно вы (подпись).

**Схема:** сначала подписываем своим закрытым ключом, затем шифруем
открытым ключом получателя. Получатель расшифровывает, затем проверяет подпись.

```bsl
// Требования:
// - Ваш сертификат (для подписи): ОтпечатокМой
// - Сертификат получателя (для шифрования): ОтпечатокПолучателя

Функция ПодписатьИЗашифровать(
    ТекстДокумента,
    ОтпечатокМой,
    ОтпечатокПолучателя
)

    Попытка
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Microsoft Strong Cryptographic Provider", "", 1
        );

        ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТекстДокумента, "UTF-8");

        // Шаг 1: подписываем своим закрытым ключом
        МойСертификат = МенеджерКрипто.ПолучитьСертификат(ОтпечатокМой);
        Подпись = МенеджерКрипто.Подписать(ДвоичныеДанные, МойСертификат);

        // Шаг 2: шифруем документ + подпись открытым ключом получателя.
        // Упаковываем оба объекта в одну структуру
        // (в реальном проекте — JSON, XML или бинарная структура)
        ДокументBase64 = Base64Строка(ДвоичныеДанные);
        ПодписьBase64  = Base64Строка(Подпись);

        Пакет = "{""doc"":""" + ДокументBase64 + """,""sig"":""" + ПодписьBase64 + """}";

        СертификатПолучателя = МенеджерКрипто.ПолучитьСертификат(ОтпечатокПолучателя);
        ПакетДвоичные = ПолучитьДвоичныеДанныеИзСтроки(Пакет, "UTF-8");

        СписокСертификатов = Новый Массив;
        СписокСертификатов.Добавить(СертификатПолучателя);

        ЗашифрованныйПакет = МенеджерКрипто.Зашифровать(
            ПакетДвоичные,
            СписокСертификатов
        );

        Сообщить("Размер зашифрованного пакета (байт): " + ЗашифрованныйПакет.Размер());
        Возврат Base64Строка(ЗашифрованныйПакет);
    Исключение
        ВызватьИсключение "Ошибка при подписании и шифровании: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции

// Получатель: расшифровываем и проверяем подпись
Процедура РасшифроватьИПроверитьПодпись(ЗашифрованныйПакетBase64)

    Попытка
        МенеджерКрипто = Новый МенеджерКриптографии(
            "Microsoft Strong Cryptographic Provider", "", 1
        );

        // Шаг 1: расшифровываем своим закрытым ключом
        ЗашифрованныйПакет = Base64Значение(ЗашифрованныйПакетBase64);
        ПакетДвоичные = МенеджерКрипто.Расшифровать(ЗашифрованныйПакет);
        Пакет = ПолучитьСтрокуИзДвоичныхДанных(ПакетДвоичные, "UTF-8");

        // Шаг 2: извлекаем документ и подпись из пакета (в реальном проекте — JSON-парсинг)
        // Здесь упрощённо, для иллюстрации схемы
        Сообщить("Пакет расшифрован. Далее — парсинг и проверка подписи из пакета.");

        // Шаг 3: проверяем подпись (см. урок 06, Пример 2)
        // МенеджерКрипто.ПроверитьПодпись(ДвоичныеДанные, Подпись);
    Исключение
        ВызватьИсключение "Ошибка при расшифровании пакета: " + ОписаниеОшибки();
    КонецПопытки;

КонецПроцедуры
```

> **В боевой системе:** замените `Microsoft Strong Cryptographic Provider` (тип `1`)
> на `Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider` (тип `80`)
> для обоих участников — подписанта и получателя.

## Задача 5. Сформировать хеш для API-запроса

Представьте записку курьеру: в конце вы дописываете секретное слово, известное
только вам и получателю. Получатель видит слово — значит, записку писали вы.
HMAC работает так же: это хеш сообщения, в который подмешан секретный ключ.
Без ключа вычислить такой хеш невозможно — именно поэтому API использует его
для проверки подлинности запроса.

**Когда нужно:** многие внешние API требуют подписи запроса через HMAC
(хеш с секретным ключом). 1С не имеет встроенной функции HMAC, но его можно
реализовать через стандартный алгоритм.

```bsl
// HMAC-SHA256: используется в AWS Signature, Telegram Bot API, большинстве REST API
// Входные данные: Сообщение (строка) и СекретныйКлюч (строка)

Функция HMACSHA256(Сообщение, СекретныйКлюч)

    Попытка
        // HMAC = H((Key XOR opad) || H((Key XOR ipad) || Message))
        // где ipad = 0x36, opad = 0x5C, блок = 64 байта
        РазмерБлока = 64;
        Ipad = 54;  // 0x36
        Opad = 92;  // 0x5C

        КлючДвоичные = ПолучитьДвоичныеДанныеИзСтроки(СекретныйКлюч, "UTF-8");

        // Нормализуем ключ до размера блока
        Если КлючДвоичные.Размер() > РазмерБлока Тогда
            Хеш = Новый ХешированиеДанных(ХешФункция.SHA256);
            Хеш.Добавить(КлючДвоичные);
            КлючДвоичные = Хеш.ХешСуммаДвоичные; // свойство, без скобок
        КонецЕсли;

        // Читаем байты ключа через БуферДвоичныхДанных — он поддерживает индексный доступ
        БуферКлюча = Новый БуферДвоичныхДанных(КлючДвоичные.Размер());
        КлючДвоичные.КопироватьВБуфер(БуферКлюча, 0);

        ЗаписьIpad = Новый ЗаписьДанных;
        ЗаписьOpad = Новый ЗаписьДанных;

        // Дополняем нулями до размера блока и XOR с ipad/opad
        Для й = 0 По РазмерБлока - 1 Цикл
            БайтКлюча = 0;
            Если й < КлючДвоичные.Размер() Тогда
                БайтКлюча = БуферКлюча[й]; // индексный доступ к байту
            КонецЕсли;
            ЗаписьIpad.ЗаписатьБайт(БитовоеИсключительноеИли(БайтКлюча, Ipad));
            ЗаписьOpad.ЗаписатьБайт(БитовоеИсключительноеИли(БайтКлюча, Opad));
        КонецЦикла;

        КлючIpad = ЗаписьIpad.ЗакрытьИПолучитьДвоичныеДанные();
        КлючOpad = ЗаписьOpad.ЗакрытьИПолучитьДвоичныеДанные();

        СообщениеДвоичные = ПолучитьДвоичныеДанныеИзСтроки(Сообщение, "UTF-8");

        // Внутренний хеш: H(ipad_key || message)
        ВнутреннийХеш = Новый ХешированиеДанных(ХешФункция.SHA256);
        ВнутреннийХеш.Добавить(КлючIpad);
        ВнутреннийХеш.Добавить(СообщениеДвоичные);
        ВнешнийВход = ВнутреннийХеш.ХешСуммаДвоичные; // свойство, без скобок

        // Внешний хеш: H(opad_key || inner_hash)
        ВнешнийХеш = Новый ХешированиеДанных(ХешФункция.SHA256);
        ВнешнийХеш.Добавить(КлючOpad);
        ВнешнийХеш.Добавить(ВнешнийВход);

        Возврат НРег(ВнешнийХеш.ХешСумма); // HEX-строка в нижнем регистре
    Исключение
        ВызватьИсключение "Ошибка при вычислении HMAC-SHA256: " + ОписаниеОшибки();
    КонецПопытки;

КонецФункции

// Пример: подписать запрос к API
СекретныйКлюч = "mysecret";
ТелоЗапроса   = "{""amount"":100,""currency"":""RUB""}";

Подпись = HMACSHA256(ТелоЗапроса, СекретныйКлюч);
Сообщить("HMAC-SHA256: " + Подпись);
// Передаётся в заголовке X-Signature или query-параметре signature
```

## Что запомнить

- **Контроль целостности файлов** — хеш SHA-256 при сохранении, сравнение при получении.
- **Хранение секретов в базе** — шифрование на техническом сертификате сервера.
  Не путать с хранением паролей пользователей — там хеш+соль.
- **Проверяйте сертификат перед использованием** — дата окончания, предупреждение
  за 30 дней. Истёкший сертификат лучше обнаружить до подписания, а не получить
  ошибку от внешней системы.
- **Подпись + шифрование** — разные инструменты для разных задач, их комбинируют:
  подписываем своим ключом, шифруем ключом получателя.
- **HMAC** — хеш с ключом для аутентификации API-запросов. В 1С реализуется
  вручную через `ХешированиеДанных`.
