# Ответы — Урок 05. Шифрование: сейф для данных

**1. Чем шифрование принципиально отличается от хеширования?**

Шифрование обратимо: зашифровали → расшифровали → исходные данные.
Хеширование необратимо: из хеша исходные данные не восстановить.

Хеширование применяют когда нужно проверить данные, не хранить их (пароли, контрольные суммы).
Шифрование применяют когда данные нужно и защитить, и потом прочитать (персональные данные, конфиденциальные документы).

---

**2. Каким ключом шифруют, каким расшифровывают?**

В 1С шифрование работает по схеме гибридного шифрования:

1. Генерируется случайный симметричный **сессионный ключ**.
2. Сами данные шифруются симметрично — это быстро.
3. Сессионный ключ шифруется **открытым ключом** получателя (из сертификата).

Расшифровка выполняется в обратном порядке:
1. Сессионный ключ расшифровывается **закрытым ключом** получателя.
2. Данные расшифровываются сессионным ключом.

Зашифровать может любой, у кого есть сертификат получателя.
Расшифровать может только тот, у кого есть соответствующий закрытый ключ.
Платформа 1С выполняет всё это автоматически через `МенеджерКрипто.Зашифровать()`.

---

**3. Почему зашифрованный файл всегда больше исходного?**

Результат шифрования в 1С — CMS-конверт (формат PKCS#7 EnvelopedData). Он содержит:
- идентификатор получателя (ссылка на сертификат)
- зашифрованный сессионный ключ (несколько сотен байт, зависит от алгоритма)
- идентификатор алгоритма
- зашифрованные данные

Это несколько сотен байт накладных расходов независимо от размера исходных данных.

---

**4. Нужно ли при расшифровке указывать сертификат или закрытый ключ явно?**

Нет. `МенеджерКрипто.Расшифровать(ЗашифрованныеДвоичные)` — единственный аргумент это сами зашифрованные данные.

Платформа читает CMS-конверт, находит внутри него идентификатор сертификата и сама ищет соответствующий закрытый ключ в личном хранилище текущего пользователя.

---

**5. Каким типом реквизита хранить зашифрованные данные?**

**Зашифрованная строка** (небольшой объём) → `Тип = Строка`, сохранять как Base64.
Итоговый размер: CMS-конверт (исходные данные + служебная часть контейнера) × 4/3 из-за Base64.
Для надёжности выбирайте `Строка неограниченная` или задавайте длину с запасом.

**Зашифрованный файл** (большой объём) → `Тип = ХранилищеЗначений`. Двоичные данные сохраняются напрямую без промежуточного Base64.

```bsl
// Строка → Base64 → Строковый реквизит
Объект.ЗашифрованныеДанные = Base64Строка(ЗашифрованныеДвоичные);

// Файл → ХранилищеЗначений
Объект.ЗашифрованныйФайл = Новый ХранилищеЗначений(ЗашифрованныеДвоичные, Новый СжатиеДанных(0));
```
