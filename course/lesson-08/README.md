# Урок 08. Диагностика: когда что-то пошло не так

> ⏱ Время: ~45 минут  
> Что нужно до начала: Уроки 01–07  
> Что будет после: умеете быстро находить причину ошибки и устранять её

## Зачем это знать

Представьте автомобиль: на панели загорелась лампочка. Это не говорит сразу,
что именно сломано — аккумулятор, топливо, датчик. Но у механика есть чеклист
и сканер, которые быстро сужают круг причин.

Криптографический код почти всегда работает, пока не перестаёт.
Сертификат истёк. Ключ «потерялся» после переустановки Windows.
Подпись не прошла проверку — хотя ничего не менялось.
Провайдер выдаёт «NTE_BAD_ALGID» (код ошибки Windows CryptoAPI: неподдерживаемый алгоритм).

Сообщения об ошибках в криптографии часто малоинформативны.
Этот урок — практический справочник: что означает ошибка и что с ней делать.

## Чеклист перед отладкой

Прежде чем разбирать конкретную ошибку — проверьте базовые вещи.
Большая часть типовых проблем находится именно здесь.

```
□ Сертификат установлен в нужном хранилище?
  → certmgr.msc → Личные → Сертификаты

□ У сертификата есть закрытый ключ?
  → Значок замочка на иконке сертификата в certmgr.msc
  → Или Пример 1 в этом уроке

□ Сертификат действителен (не истёк)?
  → Двойной клик на сертификате → вкладка Общие → «Действителен с / по»

□ Имя и тип криптопровайдера совпадают с тем, что задано в коде?
  → Пример 1 в этом уроке

□ Кодировка строки совпадает при шифровании и расшифровании?
  → Используйте UTF-8 без BOM во всех операциях курса;
    при интеграции с внешними системами — проверьте их документацию

□ Платформа 1С не ниже 8.3.6?
  → МенеджерКриптографии появился в 8.3.6
```

## Каталог ошибок

### «Не удалось найти сертификат»

**Симптом:**
```
Не удалось найти сертификат. 
Элемент не найден. (0x80092004)
```

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Неверный отпечаток | Скопируйте отпечаток из certmgr.msc, уберите пробелы |
| Сертификат в другом хранилище | `certmgr.msc` — хранилище CurrentUser; для LocalMachine используйте `certlm.msc` или оснастку MMC |
| Сертификат не установлен | Установите через certmgr.msc или PowerShell |

```bsl
// Диагностика: вывести все сертификаты в хранилище
МенеджерКрипто = Новый МенеджерКриптографии(
    "Microsoft Strong Cryptographic Provider", "", 1
);
Сертификаты = МенеджерКрипто.ПолучитьСертификаты(
    РасположениеХранилища.ЛичноеХранилище
);
Для Каждого Сертификат Из Сертификаты Цикл
    Сообщить(Сертификат.Отпечаток + " | " + Сертификат.Субъект);
КонецЦикла;
```

### «Не удалось создать подпись» / «Нет доступа к закрытому ключу»

**Симптом:**
```
Ошибка при создании подписи.
Не удалось создать подпись. Отказано в доступе. (0x80090010)
```

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Закрытый ключ отсутствует | Переустановите сертификат с закрытым ключом (.pfx) |
| Нет прав на чтение ключа | Запустите 1С от имени пользователя, под которым установлен ключ |
| Ключ на токене, токен не подключён | Подключите USB-токен (Рутокен, eToken) |

```bsl
// Проверить доступность закрытого ключа для операции подписи
МенеджерКрипто = Новый МенеджерКриптографии(
    "Microsoft Strong Cryptographic Provider", "", 1
);
Отпечаток = "A1B2C3D4E5F6...";

// Пробуем подписать тестовую строку: успех = ключ и все условия в порядке
Попытка
    Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);
    ТестДвоичные = ПолучитьДвоичныеДанныеИзСтроки("test", "UTF-8");
    МенеджерКрипто.Подписать(ТестДвоичные, Сертификат);
    Сообщить("Закрытый ключ доступен.");
Исключение
    Сообщить("Не удалось выполнить тестовую подпись: " + ОписаниеОшибки());
    // Причины: сертификат не найден, нет ключа, нет прав, токен без PIN, провайдер
КонецПопытки;
```

### «Не удалось расшифровать»

**Симптом:**
```
Не удалось расшифровать.
Указанный сертификат не найден. (0x8009200C)
```

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Данные зашифрованы для другого сертификата | Расшифровать может только владелец того сертификата, которым шифровали |
| Закрытый ключ сертификата удалён | Восстановить из резервной копии `.pfx`; если копий не осталось — штатное восстановление невозможно |
| Данные зашифрованы другим провайдером | Используйте тот же провайдер, которым шифровали |

> **Важно:** если не осталось ни одной копии закрытого ключа, расшифровать данные
> практически невозможно. Используйте разрешённый вашей политикой способ резервного
> хранения: `.pfx`-файл, токен или регламент удостоверяющего центра.

### «Подпись не прошла проверку»

**Симптом:**
```
Подпись не прошла проверку.
```

**Причины:**

1. **Документ изменился после подписания** — добавился невидимый символ,
   изменилась кодировка, файл открыли в текстовом редакторе и он добавил BOM.

2. **Несовпадение кодировки** — документ подписывали в UTF-8, проверяют в Windows-1251.

3. **Подпись от другого документа** — перепутали файл подписи.

```bsl
// Диагностика: вычислить хеш документа в двух точках и сравнить
Функция ХешДокумента(ТекстДокумента)
    ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки(ТекстДокумента, "UTF-8");
    Хеш = Новый ХешированиеДанных(ХешФункция.SHA256);
    Хеш.Добавить(ДвоичныеДанные);
    Возврат НРег(Хеш.ХешСумма);
КонецФункции

// Вызвать в точке подписания и в точке проверки.
// Если хеши разные — документ изменился.
Сообщить("Хеш: " + ХешДокумента(ТекстДокумента));
```

### «Неверный алгоритм» / «NTE_BAD_ALGID»

**Симптом:**
```
Криптографический провайдер не поддерживает данный алгоритм. (0x80090008)
```

**Причина:** провайдер Microsoft Strong CSP не поддерживает алгоритм операции
(например, SHA-256 для подписи — нужен тип 24, а не 1).

**Решение:**

| Задача | Провайдер | Тип |
|--------|-----------|-----|
| RSA + SHA-1 | Microsoft Strong Cryptographic Provider | 1 |
| RSA + SHA-256 | Microsoft Enhanced RSA and AES Cryptographic Provider | 24 |
| ГОСТ Р 34.10-2012 | Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider | 80 |

```bsl
// Для операций с SHA-256 используйте тип 24
МенеджерКрипто = Новый МенеджерКриптографии(
    "Microsoft Enhanced RSA and AES Cryptographic Provider", "", 24
);
```

> **В боевой системе:** для КЭП и ГОСТ-подписей используйте КриптоПро CSP (тип `80`).
> Microsoft-провайдеры (тип `1` и `24`) подходят только для учебных RSA-примеров.

### «Истёкший сертификат»

**Симптом:**
```
Срок действия сертификата истёк. (0x800B0101)
```

Сертификат нужно перевыпустить. Для тестов — создать новый самоподписанный
(инструкция в docs/certificates.md). Для боевой КЭП — обратиться в УЦ.

## Пример 1. Диагностика провайдеров

Если не уверены в названии и типе установленного провайдера:

```bsl
Попытка
    МенеджерКрипто = Новый МенеджерКриптографии;
    Провайдеры = МенеджерКрипто.ПолучитьДоступныеПровайдеры();
    Если Провайдеры.Количество() = 0 Тогда
        Сообщить("Провайдеры не обнаружены.");
    КонецЕсли;
    Для Каждого Провайдер Из Провайдеры Цикл
        Сообщить("Тип: " + Провайдер.Тип + " | " + Провайдер.Имя);
    КонецЦикла;
Исключение
    Сообщить("Ошибка: " + ОписаниеОшибки());
КонецПопытки;
```

## Пример 2. Полная диагностика сертификата

```bsl
// Вывести всё что важно знать о сертификате перед использованием

Процедура ДиагностикаСертификата(МенеджерКрипто, Отпечаток)

    Попытка
        Сертификат = МенеджерКрипто.ПолучитьСертификат(Отпечаток);
    Исключение
        Сообщить("✗ Сертификат не найден по отпечатку: " + Отпечаток);
        Сообщить("  Причина: " + ОписаниеОшибки());
        Возврат;
    КонецПопытки;

    Сообщить("Сертификат найден:");
    Сообщить("  Субъект:   " + Сертификат.Субъект);
    Сообщить("  Издатель:  " + Сертификат.Издатель);
    Сообщить("  Начало:    " + Формат(Сертификат.ДатаНачала,    "ДЛФ=Д"));
    Сообщить("  Окончание: " + Формат(Сертификат.ДатаОкончания, "ДЛФ=Д"));

    Сейчас = ТекущаяДата();
    Если Сейчас < Сертификат.ДатаНачала Тогда
        Сообщить("  [!] Сертификат ещё не начал действовать");
    ИначеЕсли Сейчас > Сертификат.ДатаОкончания Тогда
        Сообщить("  [!] Сертификат ИСТЁК");
    Иначе
        ДнейОсталось = Цел((Сертификат.ДатаОкончания - Сейчас) / 86400);
        Если ДнейОсталось < 30 Тогда
            Сообщить("  [!] Сертификат истекает через " + ДнейОсталось + " дн.");
        Иначе
            Сообщить("  [+] Сертификат действителен, осталось " + ДнейОсталось + " дн.");
        КонецЕсли;
    КонецЕсли;

    // Проверяем возможность подписи (ключ, права, провайдер, алгоритм)
    Попытка
        ТестДвоичные = ПолучитьДвоичныеДанныеИзСтроки("test", "UTF-8");
        МенеджерКрипто.Подписать(ТестДвоичные, Сертификат);
        Сообщить("  [+] Закрытый ключ доступен");
    Исключение
        Сообщить("  [!] Закрытый ключ НЕДОСТУПЕН: " + ОписаниеОшибки());
    КонецПопытки;

КонецПроцедуры

// Использование
МенеджерКрипто = Новый МенеджерКриптографии(
    "Microsoft Strong Cryptographic Provider", "", 1
);
ДиагностикаСертификата(МенеджерКрипто, "A1B2C3D4E5F6...");
```

## Полезные инструменты вне 1С

**certmgr.msc** — просмотр хранилища сертификатов Windows.
`Win+R` → `certmgr.msc`. Замочек на иконке = есть закрытый ключ.

**PowerShell: список сертификатов с ключами:**
```powershell
Get-ChildItem Cert:\CurrentUser\My |
  Where-Object { $_.HasPrivateKey } |
  Select-Object Subject, Thumbprint, NotAfter
```

**PowerShell: создать новый тестовый сертификат:**
```powershell
$cert = New-SelfSignedCertificate `
    -Subject "CN=CryptoLearn" `
    -CertStoreLocation "Cert:\CurrentUser\My" `
    -KeyAlgorithm RSA -KeyLength 2048 `
    -KeyUsage KeyEncipherment, DigitalSignature `
    -Provider "Microsoft Enhanced RSA and AES Cryptographic Provider" `
    -NotAfter (Get-Date).AddYears(2)
Write-Host "Отпечаток: $($cert.Thumbprint)"
# Провайдер тип 24 (Enhanced RSA and AES) — совместим с SHA-256 в 1С
```

**Просмотр CMS-файлов** — утилита `certutil.exe` (встроена в Windows):
```
certutil -dump Договор.pdf.sig
certutil -dump зашифрованный.p7m
```

## Что запомнить

- **Начинайте с чеклиста:** хранилище, наличие закрытого ключа, срок действия,
  имя провайдера, кодировка. 80% проблем — там.
- **Резервная копия ключа критична.** Если копий не осталось — зашифрованные
  данные восстановить практически невозможно. Используйте разрешённый политикой
  способ: `.pfx`, токен или регламент УЦ.
- **Кодировка имеет значение.** Строка в UTF-8 и в Windows-1251 — разные байты,
  разный хеш, разная подпись. Фиксируйте UTF-8 без BOM везде.
- **Тип провайдера важен.** Microsoft Strong CSP (тип 1) — SHA-1, тип 24 — SHA-256,
  КриптоПро (тип 80) — ГОСТ Р 34.10-2012.
- **`certutil -dump`** — быстро посмотреть структуру многих CMS-файлов без написания кода.
