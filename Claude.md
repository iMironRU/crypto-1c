# Claude.md — Инструкции для ИИ-ассистента

## Что это за проект

Мини-курс «Криптография в 1С» — учебный материал для разработчиков 1С,
которые знают язык, но никогда не работали с криптографией.

Сопровождается демонстрационной конфигурацией 1С с рабочими примерами кода,
которые читатель может запустить и использовать в своих проектах.

## Аудитория

Разработчик 1С с опытом от 1 года. Знает язык, работал с документами,
справочниками, запросами. Криптография — новая тема. Слова «хеш», «сертификат»,
«ключ» слышал, но не понимает как это работает и зачем нужно.

## Фундаментальный принцип: всё — двоичные данные

Это самое важное, что читатель должен понять до первого примера.

**Криптография работает только с байтами.** Не со строками, не с числами,
не с датами — только с последовательностями байт (`ДвоичныеДанные`).
Любой текст перед криптографической операцией превращается в байты.
Результат любой операции — тоже байты.

Это означает два обязательных шага в каждом примере:

```
Строка → ДвоичныеДанные  →  [крипто-операция]  →  ДвоичныеДанные → Строка/Хранилище
```

**Правило для всех уроков и примеров кода:**
- Каждый пример, где на вход идёт текст, явно показывает преобразование
  строки в `ДвоичныеДанные` перед операцией.
- Каждый пример, где результат — двоичные данные, явно показывает
  как их представить (Base64, HEX) и как сохранить в базе.
- Нельзя «прятать» это внутри функции без объяснения — читатель
  должен видеть и понимать каждый шаг.

Подробно: `docs/binary-data.md`

## Правила написания текстов

**Тон:** дружелюбный, прямой. Не «рекомендуется применять», а «используйте».
Не «следует отметить», а «важно».

**Аналогии:** каждую новую концепцию вводить через бытовую аналогию
*до* технического объяснения. Аналогия — не украшение, а точка входа.

**Код:** каждый пример должен быть воспроизводимым. Если для запуска
нужны предварительные шаги (установить сертификат, создать файл) —
описать их явно, пошагово, прямо перед кодом.

**Примечания ГОСТ:** когда пример работает через Microsoft CSP,
добавлять блок «В боевой системе» с указанием что именно меняется
при переходе на КриптоПро. Одно место, одна строка.

**Длина раздела:** ~300–500 слов на смысловой блок. Если больше —
делить на подразделы.

## Правила написания кода

**Язык:** встроенный язык 1С (BSL), русские ключевые слова.

**Комментарии:** каждая нетривиальная строка — с комментарием.
Код должен читаться как псевдокод.

**Переменные:** имена на русском, описательные.
`ХешДвоичные` а не `х` или `result`.

**Обработка ошибок:** все примеры с `Попытка/Исключение`,
с понятным сообщением об ошибке.

**Файлы кода:** расширение `.bsl`. Имя файла — kebab-case,
описывает что делает код: `hash-password-with-salt.bsl`.

**Воспроизводимость:** в начале каждого файла кода —
блок комментариев с требованиями:

```
// Требования:
// - Платформа 1С 8.3.19+
// - Windows (Microsoft Strong Cryptographic Provider встроен в ОС)
// - Для ГОСТ: КриптоПро CSP 5.x
```

## Важные решения

- **Двоичные данные — явно.** Каждый пример показывает полный путь:
  строка → `ДвоичныеДанные` → операция → результат → хранение в БД.
  Конвертации не прячутся внутри функций без объяснения.
- **Без платных компонент.** Все примеры работают на встроенном
  Microsoft Strong Cryptographic Provider (Windows).
- **ГОСТ — отдельно.** Переход на КриптоПро описывается как
  отдельный шаг, не перемешивается с основными примерами.
- **Конфигурация пишется после курса.** Сначала текст —
  потом видно какие модули нужны.
- **Код = уроки.** Файлы в `/code` — это те же примеры что
  в уроках, просто вырезанные для удобного копирования.
