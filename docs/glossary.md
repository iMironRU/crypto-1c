# Словарь терминов

Краткий справочник. Если термин встречается в уроке и непонятен — ищите здесь.

---

## Б

**Base64** — способ представить двоичные данные в виде текста из 64 символов
(A–Z, a–z, 0–9, +, /). Увеличивает размер данных примерно в 1.33 раза.
Используется для хранения хешей и подписей в строковых реквизитах базы данных,
передачи в JSON, HTTP-заголовках.
*Пример: `Base64Строка(ДвоичныеДанные)` → `"47DEQpj8HBSa+/T..."`*

**BOM (Byte Order Mark)** — невидимые байты в начале текстового файла,
обозначающие кодировку. В UTF-8 BOM выглядит как три байта `EF BB BF`.
Присутствие BOM меняет хеш данных — для криптографии всегда
используйте UTF-8 **без BOM**.



**Асимметричное шифрование** — способ шифрования с двумя ключами:
открытым (для шифрования) и закрытым (для расшифровки).
Что зашифровано одним — расшифровывается только вторым.
*Аналогия: открытый замок без ключа — любой может закрыть, открыть только владелец ключа.*

---

## Д

**ДвоичныеДанные** — тип платформы 1С для хранения последовательности байт.
Все криптографические операции принимают и возвращают именно этот тип.
Строки, числа, файлы — всё должно быть приведено к `ДвоичныеДанные`
перед передачей в криптографические функции.

---

## Г

**ГОСТ** — российские государственные стандарты криптографии.
ГОСТ Р 34.10-2012 — электронная подпись.
ГОСТ Р 34.11-2012 (Стрибог) — хеширование.
ГОСТ Р 34.12-2015 (Кузнечик) — симметричное шифрование.
Обязательны для работы с госсистемами и для КЭП.

---

## З

**Закрытый ключ (приватный ключ)** — секретная часть ключевой пары.
Никому не передаётся. Используется для подписания документов
и расшифровки данных.

---

## К

**КЭП (Квалифицированная электронная подпись)** — электронная подпись,
выданная аккредитованным удостоверяющим центром после личного визита.
Равна собственноручной подписи по закону (63-ФЗ).

**Криптопровайдер (СКЗИ)** — программа, которая выполняет
криптографические операции. Примеры: КриптоПро CSP, ViPNet CSP,
Microsoft Strong Cryptographic Provider.

---

## О

**Открытый ключ (публичный ключ)** — публичная часть ключевой пары.
Свободно распространяется. Используется для проверки подписи
и шифрования данных для владельца.

**Отпечаток сертификата** — хеш-сумма файла сертификата.
Используется как идентификатор сертификата в коде 1С.

---

## С

**Сертификат** — файл (.cer, .pfx), который содержит открытый ключ,
данные о владельце и подпись удостоверяющего центра.
*Аналогия: паспорт для открытого ключа — подтверждает кто владелец.*

**Симметричное шифрование** — способ шифрования, где один и тот же
ключ используется для шифрования и расшифровки.
Быстрее асимметричного, но требует безопасной передачи ключа.

**Соль (Salt)** — случайная строка, добавляемая к паролю перед хешированием.
Делает хеши одинаковых паролей разными у разных пользователей.
Защищает от атак по словарю.

---

## У

**УЦ (Удостоверяющий центр)** — организация, которая выдаёт сертификаты
после проверки личности владельца.
Для КЭП — только аккредитованные УЦ (список на сайте Минцифры).

---

## Х

**Хеш (хеш-сумма, дайджест)** — результат необратимого преобразования
данных в строку фиксированной длины.
*Аналогия: отпечаток пальца документа — уникален, нельзя восстановить документ по отпечатку.*

**Хеш-функция** — алгоритм вычисления хеша (SHA256, SHA512, MD5 и др.).

---

## Ц

**Цепочка сертификатов** — последовательность сертификатов от
конечного пользователя до корневого УЦ. Используется для проверки
достоверности сертификата.

---

## Э

**ЭЦП / ЭП (Электронная подпись)** — результат криптографического
преобразования, подтверждающий авторство и целостность документа.
Не шифрует документ — только подтверждает кто и когда подписал.

## Английские термины

| Термин | Перевод | Объяснение |
|--------|---------|-----------|
| AES | — | Алгоритм симметричного шифрования (не ГОСТ) |
| Base64 | — | Способ представить двоичные данные в виде текста |
| CAdES | — | Формат электронной подписи для файлов |
| CRL | Список отозванных сертификатов | Публикуется УЦ, проверяется при верификации подписи |
| HEX | Шестнадцатеричное | Представление двоичных данных символами 0–9, a–f. SHA256 в HEX = 64 символа. Удобен для логов и сравнения с внешними утилитами. |
| HTTPS | — | HTTP поверх TLS; шифрует канал передачи |
| JWT | JSON Web Token | Формат токена для аутентификации в API |
| MD5 | — | Устаревший алгоритм хеширования, взломан |
| mTLS | Mutual TLS | TLS с взаимной аутентификацией клиента и сервера |
| OCSP | — | Протокол онлайн-проверки статуса сертификата |
| PAdES | — | Формат электронной подписи внутри PDF |
| PKCS#7 / CMS | — | Формат контейнера для подписанных/зашифрованных данных |
| RSA | — | Алгоритм асимметричного шифрования (не ГОСТ) |
| SHA256 | — | Современный алгоритм хеширования, безопасен |
| SHA512 | — | Современный алгоритм хеширования, безопасен, длиннее SHA256 |
| TLS | — | Протокол шифрования канала связи |
| TSP | Метка времени | Подтверждает момент создания подписи |
| XAdES | — | Формат электронной подписи для XML |
