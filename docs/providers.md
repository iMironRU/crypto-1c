# Справочник криптопровайдеров

Быстрая справка: какой провайдер использовать в коде 1С.

---

## Для учебных примеров (без установки)

```bsl
// Microsoft Strong Cryptographic Provider — тип 1 (PROV_RSA_FULL)
// Встроен в Windows, ничего устанавливать не нужно
// Поддерживает: RSA, SHA-1. Используется в примерах уроков 01–06.
// Для SHA-256 нужен тип 24 (Microsoft Enhanced RSA and AES Cryptographic Provider).
МенеджерКрипто = Новый МенеджерКриптографии(
    "Microsoft Strong Cryptographic Provider",
    "",
    1  // тип провайдера
);
```

---

## Для боевых систем с ГОСТ (КриптоПро 5.x)

```bsl
// Тип 80 — ГОСТ Р 34.10-2012 / 256 бит (актуальный стандарт КриптоПро 4.x–5.x)
// Тип 75 — старый ГОСТ Р 34.10-2001, не используйте для новых проектов
МенеджерКрипто = Новый МенеджерКриптографии(
    "Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider",
    "",
    80  // тип провайдера
);
```

| Параметр | Значение |
|----------|---------|
| Имя модуля | `Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider` |
| Тип модуля | `80` (ГОСТ-2012/256); `81` (ГОСТ-2012/512) |
| OID открытого ключа | `1.2.643.7.1.1.1.1` |
| OID алгоритма подписи | `1.2.643.7.1.1.3.2` |
| OID алгоритма хеша | `1.2.643.7.1.1.2.2` |

## Как проверить что провайдер установлен

**Через реестр (все провайдеры):**
```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Cryptography\Defaults\Provider
```

**Через КриптоПро CSP:** Пуск → КриптоПро CSP → вкладка «Оборудование».

**Через 1С (список провайдеров):**
```bsl
// Выводит все установленные провайдеры
МенеджерКрипто = Новый МенеджерКриптографии;
Для Каждого Провайдер Из МенеджерКрипто.ПолучитьДоступныеПровайдеры() Цикл
    Сообщить(Провайдер.Имя + " (тип: " + Провайдер.Тип + ")");
КонецЦикла;
```

## Частые ошибки при инициализации

| Ошибка | Причина | Решение |
|--------|---------|---------|
| «Провайдер не определён» | Неправильное имя или тип | Сверьте с таблицами выше, имя чувствительно к регистру |
| «Криптопровайдер не установлен» | КриптоПро не установлен | Установите КриптоПро CSP с официального сайта |
| «Нет связи ключа с сертификатом» | Сертификат установлен без закрытого ключа | Устанавливайте через КриптоПро CSP, не через certmgr |
